# Content Project Template

## Folder Structure

```
ACTIVE_PROJECTS\[project-name]\
├── CLAUDE.md              ← Fill from _template_project_claude.md
├── brief.md               ← Content goal, audience, platform
├── ideas\
│   └── [topic].md         ← Raw ideas and angles
├── scripts\
│   └── YYYY-MM-DD_[title]_draft.md
├── drafts\
│   └── YYYY-MM-DD_[title]_draft.md
└── finals\
    └── YYYY-MM-DD_[title]_final.md
```

---

---

## Session Startup (Content)

1. Read project CLAUDE.md
2. Read content_operating_system.md
3. Read voice_constraints.md
4. If pillar-driven content: read content_pillars.md

---

## Content Output Standards

Ideas:
- Bullet format, raw capture, no polish needed
- Capture angle, hook, and core insight — not the full execution

Scripts / drafts:
- Named with date + title + stage
- Include: hook, structure, key points, CTA

Finals:
- Reviewed against voice_constraints.md before marking final
- Stored in `finals\` with `_final` suffix
