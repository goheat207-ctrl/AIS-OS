# Thinking Frameworks

## Purpose

These are structured ways to approach unclear problems.
Load this file when the task requires reasoning, not just execution.

---

## Framework 1 — First Principles

Use when: conventional approaches aren't working, or you're starting from scratch.

```
1. State the goal precisely
2. List what you know to be true (facts, not assumptions)
3. List what you are assuming (explicitly)
4. Build the solution from the facts only
5. Test assumptions against the solution
```

Anti-pattern: copying existing approaches without questioning whether they apply.

---

## Framework 2 — Inversion

Use when: you can't see how to succeed, but you can see how to fail.

```
1. Define what failure looks like in detail
2. List everything that causes that failure
3. Eliminate or design around each failure cause
4. The remaining path is more likely to succeed
```

Example for trading: instead of "how do I make money?" ask "how do I definitely lose money?" and avoid all of those things.

---

## Framework 3 — Separation of Concerns

Use when: a problem feels too large or tangled to solve.

```
1. List every distinct concern in the problem
2. Draw a boundary around each concern
3. Solve each concern independently
4. Define how the solutions interface
```

This is how good systems are built and how complex problems become manageable.

---

## Framework 4 — Pre-Mortem

Use when: about to commit to a plan or decision.

```
1. Assume the plan failed completely
2. Write a story explaining how it failed
3. Identify the most plausible failure modes
4. Adjust the plan to address them
```

The goal is not to find reasons to quit. It is to find solvable weaknesses before they surface.

---

## Framework 5 — Minimum Viable Clarity

Use when: you feel like you need more information before starting.

```
1. What is the minimum you need to know to take a useful first step?
2. Take that step.
3. Gather information from the result.
4. Repeat.
```

Most "I need more information" feelings are analysis paralysis.
The cheapest way to get information is often to act and observe.

---

## Choosing a Framework

| Situation                           | Use                         |
| ----------------------------------- | --------------------------- |
| Starting something new from scratch | First Principles            |
| Stuck and can't see the path        | Inversion                   |
| Problem is tangled and complex      | Separation of Concerns      |
| About to commit to something        | Pre-Mortem                  |
| Waiting for information to start    | Minimum Viable Clarity      |
