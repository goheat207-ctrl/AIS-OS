# Risk Framework

## Core Rule

Define the risk before entering.
If you can't define the loss, don't take the trade.

---

## Position Sizing

```
Risk per trade = Account size × Risk % per trade
Position size  = Risk per trade ÷ (Entry price − Stop price)
```

Example:
- Account: $50,000
- Risk per trade: 1% = $500
- Entry: $10.00, Stop: $9.50 ($.50 risk per share)
- Position size: $500 ÷ $0.50 = 1,000 shares

Adjust risk % based on setup quality:
- A-setup: up to full risk %
- B-setup: 50-75% of full risk %
- C-setup or uncertain: pass or minimal size

---

## Daily Loss Limit

Set a maximum daily loss as a % of account.
When hit: stop trading for the day. No exceptions.

Recommended starting point: 2-3× your average risk per trade.

Example: if you risk $500/trade, daily loss limit might be $1,000-$1,500.

---

## Red Day Protocol

When you hit the daily loss limit:
1. Stop all trading immediately
2. Review: was it bad luck or bad execution?
3. Journal the day before next session
4. Consider reduced size the following session

---

## Sizing Rules

Never:
- Average down into a losing position
- Add to a position that hasn't proven itself
- Size up after a losing streak to "make it back"
- Size up out of overconfidence after a winning streak

Always:
- Size based on the stop, not on conviction
- Reduce size when in drawdown
- Start small when trading a new setup or scanner

---

## Risk-Reward

Minimum target before entry: 2:1 reward-to-risk.
Preferred: 3:1 or better.

If the technical setup doesn't offer 2:1 to a logical target: pass.

---

## Risk Framework Maintenance

Review this framework quarterly or after a significant drawdown.
Update based on journal findings.

Rules that aren't followed should either be:
- Fixed (made more specific and actionable)
- Removed (if they don't reflect reality)

Not: kept as aspirational guidelines that get ignored.
