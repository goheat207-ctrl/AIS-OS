# Trading Operating System

## What This Is

This is the master context file for all trading work in AI_OS.
Load this at the start of any trading-domain session.

---

## Trading Philosophy

The market is a wealth transfer system.
Money moves from undisciplined to disciplined.
The edge is not information — it is execution and behavior.

---

## Core Operating Principles

1. **Process over outcome.** A good trade can lose. A bad trade can win. Judge the process.
2. **Risk first.** Define the loss before entering. Always.
3. **Asymmetry matters.** Only take trades where the potential reward justifies the risk.
4. **No trade is required.** Sitting out is a valid decision.
5. **Mistakes are data.** A documented mistake is tuition paid. An undocumented one is wasted.

---

## Daily Operating Structure

```
Pre-market:
  - Review watchlist and catalysts
  - Identify key levels
  - Set risk parameters for the day

Market open:
  - Execute only A-setups
  - No chasing
  - No averaging down into losers

Midday:
  - Assess morning performance
  - Adjust plan or stop trading

Close / Recap:
  - Journal every trade taken (and notable ones passed)
  - Identify mistakes, improvements, A-setups
```

## AI Role in Trading Work

| Task                   | AI Responsibility                            |
| ---------------------- | -------------------------------------------- |
| Scanner development    | Logic design, ThinkScript, criteria refinement |
| Journal review         | Pattern identification, mistake categorization |
| Research               | Catalyst analysis, float/dilution research    |
| Recap writing          | Structured output of trade data               |
| Rule development       | Synthesizing journal patterns into rules      |

AI does NOT:
- Make entry/exit decisions
- Override trader judgment
- Generate buy/sell signals
